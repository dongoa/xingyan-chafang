# 星衍茶坊小程序外壳（毕设演示用）

## 是什么

一个最简的微信小程序，只有一个页面，页面里放一个 `<web-view>` 指向本地运行的 H5 原型（`http://localhost:5176/agree`）。用于毕设答辩时在笔记本上通过微信开发者工具模拟器演示完整原型。

## 怎么跑

### 1. 先启动 H5

```bash
cd ../app
npm run dev
# 保持这个终端开着，看到 VITE ready 且 Local: http://localhost:5176/ 即可
```

### 2. 再打开小程序

1. 安装 **微信开发者工具**（https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html）
2. 打开 → "导入项目" → 目录选到 `miniapp/`（本目录）
3. **AppID**：
   - 如果客户已把 AppID 给你 → 填进 `project.config.json` 的 `"appid"` 字段（现在占位是 `touristappid`）
   - 没有 → 选 "使用测试号" 也能跑
4. 导入成功后进入项目，右上角 **"详情" → "本地设置"**，勾上这三个：
   - ✅ **不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书**
   - ✅ ES6 转 ES5
   - ✅ 增强编译
5. 点击左上角 "编译"，模拟器里会加载出完整 H5 原型

### 3. 关键注意

- **web-view 组件官方只支持企业/政府主体小程序**。如果客户注册的是个人小程序，真机不可用；**但开发者工具模拟器在勾选"不校验合法域名"后通常能显示**，答辩现场演示就走这条路。
- **URL 必须是 HTTP 协议**，且与开发者工具在同一台电脑上（因为是 `localhost`）。
- 如果想用手机扫开发者工具二维码真机预览，要把 `preview.js` 里的 `localhost` 改成电脑的局域网 IP（如 `192.168.1.246`），且手机和电脑同 WiFi——但个人主体 + 未备案域名这条依然卡真机，仅模拟器可行。

## 故障排查

| 症状 | 原因 | 解决 |
|---|---|---|
| web-view 空白 | H5 没启动 | 确认 `npm run dev` 在跑 |
| "请在小程序管理后台配置业务域名" | 没勾豁免 | 打开"不校验合法域名"开关 |
| "web-view 组件不支持" | 个人主体小程序 | 换企业主体 AppID；或仅用开发者工具模拟器 |
| 404/白屏 | 端口不是 5176 | 改 `preview.js` 里的 `url` |

## 自动化检查

H5 侧新增了自动化测试，用来模拟微信小程序内嵌环境。当前覆盖 77 条用例：

- H5 自画胶囊是否已隐藏
- 页面根节点是否避开微信顶部安全区
- 是否出现横向滚动或真实文字/交互元素越界
- 每个关键路由是否能在 375×812 移动视口下渲染且没有运行时报错
- P0 预约链路：同意条款 → 登录 → 首页 → 预约 → 支付成功
- 商城链路：商品详情 → 购物车 → 支付成功
- 底部 TabBar：首页 / 茶文化 / 商城 / 会员中心是否都有真实页面承接

运行：

```bash
cd ../app
npm run test:webview
```

需要看浏览器过程时运行：

```bash
cd ../app
npm run test:webview:headed
```

真实微信开发者工具模拟器仍需人工打开 `miniapp/` 验证 WebView 视觉效果。
